# 專案記憶系統 — Memory

> **每次新對話必須先讀取此目錄下的檔案，恢復專案記憶。**

## 機制說明

AI 沒有跨對話記憶，因此所有重要決策、進度、變更都必須**寫入檔案**持久化。
每次新對話的第一步是讀取 `memory/` 目錄，重建上下文。

## 記憶檔案

| 檔案 | 用途 | 更新時機 |
|------|------|---------|
| `memory/changelog.md` | 所有變更的時間線紀錄 | 每次修改 spec、寫程式碼、做決策後 |
| `memory/decisions.md` | 架構決策紀錄（ADR） | 遇到衝突或重要選擇時 |
| `memory/dev-status.md` | 開發現況快照 | 每次工作結束前更新 |
| `memory/backlog.md` | 待辦事項與未來點子 | 使用者提出新想法但暫不執行時 |

## 寫入規則

### changelog.md
每筆紀錄格式：
```markdown
### [YYYY-MM-DD] 標題
- **觸發者**：使用者需求 / 衝突解決 / 自發優化
- **影響範圍**：列出受影響的 spec / 程式檔案
- **變更內容**：簡述做了什麼
- **相關決策**：ADR-XXX（如果有的話）
```

### decisions.md
每筆決策格式（ADR = Architecture Decision Record）：
```markdown
### ADR-XXX: 標題
- **狀態**：✅ 已定案 / 🔄 可能重審 / ❌ 已廢棄
- **背景**：為什麼需要做這個決策
- **選項**：
  - A) ...
  - B) ...
- **決定**：選 X，因為...
- **影響**：修改了哪些 spec / 程式碼
```

### dev-status.md
快照格式：
```markdown
## 截至 YYYY-MM-DD 的開發狀態

### 已完成
- [x] ...

### 進行中
- [ ] ...

### 下一步
- ...

### 已知問題
- ...
```

## AI 行為準則

1. **每次工作結束前**，必須更新 `memory/dev-status.md`
2. **每次修改 spec**，必須在 `memory/changelog.md` 新增一條紀錄
3. **每次做重要決策**（選 A 不選 B），必須寫 `memory/decisions.md`
4. **使用者提到的新想法**如果現在不做，放入 `memory/backlog.md`
5. **新對話開始時**，讀取 `memory/` → 告訴使用者「我已恢復上次的專案記憶」並簡述現況
