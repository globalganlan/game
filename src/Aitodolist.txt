iOS 的 Safari 對 PWA 的記憶體限制比原生 APP 更嚴格，當 PWA 試圖載入過大或過多檔案（如大量高解析度圖片、大型 JS 檔案），瀏覽器為了維持系統穩定，會強制終止占用過多記憶體（RAM）的網頁

iOS 的 PWA（Safari WebView）記憶體上限遠低於原生 App，3D + 多個 .glb 很容易直接被系統 kill（閃退）

絕對不要「同時載入多個 GLB」
改成：

👉 用到才載（lazy load）
👉 用完就 dispose

scene.remove(model)
model.traverse(obj => {
  if (obj.geometry) obj.geometry.dispose()
  if (obj.material) {
    if (obj.material.map) obj.material.map.dispose()
    obj.material.dispose()
  }
})

不要 preload 全部資源

很多人會這樣：

Promise.all([
  loadModel(a),
  loadModel(b),
  loadModel(c),
])

👉 在 iOS = 自殺

改成：

await loadModel(a)
// 用完 unload
await loadModel(b)

WebGL context 也會爆

iOS 有時候會：

👉 WebGL context lost → 直接閃退

你要監聽：

canvas.addEventListener("webglcontextlost", (e) => {
  e.preventDefault()
  console.log("context lost")
})
