> 最後更新：2026-03-17

---

| 步驟 | 內容 | 工具 / 指令 | 產出 |
|------|------|------------|------|
| 1 | 設計英雄 | 手動 / AI 策劃 | 英雄規格表 |
| 2 | 寫入 D1 資料 | `wrangler d1 execute --remote` | 6 張表新增記錄 |

## 二、步驟 1：設計英雄

### 需要決定的欄位

| 欄位 | 範例 | 說明 |
|------|------|------|
| heroId | 16 | 流水號（從 heroes 表取 MAX+1） |
| name | 影行者 | 繁中名稱 |
| type | 敏捷 | 力量 / 敏捷 / 智慧 / 平衡 |
| rarity | SR | N / R / SR / SSR |
| baseHP | 85 | 基礎生命（一般範圍 60~140） |
| baseATK | 40 | 基礎攻擊（一般範圍 15~50） |
| baseDEF | 15 | 基礎防禦（一般範圍 10~30） |
| baseSPD | 12 | 基礎速度（一般範圍 8~15） |
| critRate | 5 | 暴擊率 %（預設 5） |
| critDmg | 50 | 暴擊傷害 %（預設 50） |
| modelId | zombie_16 | 3D 模型資料夾名稱 |
| description | ... | 英雄介紹文字 |

### 技能設計

每位英雄需要 **1 主動 + 4 被動** 技能，各有：
- **skillId**: `ACT_{heroId}` / `PAS_{heroId}_{1~4}`
- **name**: 繁中技能名
- **type**: `active` / `passive`
- **target**: 見 EffectTarget 類型
- **description**: 技能說明文字
- **icon**: emoji 字元（🔥 ❄️ 💀 🩸 etc.）
- **passive_trigger**: 被動使用（always / on_attack / on_kill / ...）

### 效果設計（v2.0 格式）

每個技能的效果寫到 `effect_templates` + `skill_effects`：
- **effectId**: `EFF_{CATEGORY}_{heroId}_{skillSuffix}_{sortOrder}`
  - 例：`EFF_DMG_16_ACT_1`（英雄16主動技的傷害效果）
- **category**: 19 種之一（damage / dot / heal / buff / debuff / cc / ...）
- **trigger_type**: 23 種之一
- **dependsOn**: 前置效果 effectId（可選）

### 稀有度 → 數值參考

| 稀有度 | HP 範圍 | ATK 範圍 | 技能複雜度 |
|--------|---------|----------|-----------|
| N | 60~80 | 15~25 | 單純（1~2 效果/技能） |
| R | 75~100 | 20~35 | 中等（2~3 效果/技能） |
| SR | 85~120 | 30~45 | 複雜（2~4 效果/技能） |
| SSR | 100~140 | 35~50 | 高階（3~5 效果/技能） |

## 三、步驟 2：寫入 D1 資料

### 涉及的 6 張表

```
1. heroes           — 英雄基礎資料
2. skill_templates   — 技能模板（5 筆：1 主動 + 4 被動）
3. hero_skills       — 英雄↔技能關聯
4. effect_templates  — 效果模板（每技能 1~5 筆）
5. skill_effects     — 技能↔效果關聯（含 dependsOn / overrideParams）
6. hero_shards       — 碎片設定（可選）
```

### SQL 範本

```sql

  ('PAS_16_1', '暗影迴避', 'passive', 'self', '永久提升閃避', '[]', 'always', '👻'),
  ('PAS_16_2', '致命突襲', 'passive', 'single_enemy', '暴擊時追加傷害', '[]', 'on_crit', '⚡'),
  ('PAS_16_3', '無影步', 'passive', 'self', '每 2 回合加速', '[]', 'every_n_turns', '🌪️'),

```

### 執行指令

```powershell
# 單筆執行
npx wrangler d1 execute globalganlan-db --remote --command "INSERT INTO ..."

# 批次執行（PowerShell 陣列）
@("INSERT ...", "INSERT ...") | ForEach-Object {
  npx wrangler d1 execute globalganlan-db --remote --command $_ 2>&1 | Select-String "Executed"
}
```

> ⚠️ `--file` 不可靠（多語句可能卡住），建議用 `--command` 單筆執行。

## 四、步驟 3：準備 3D 模型

### 方案 A：全新模型（推薦）

1. 取得 FBX 模型（Mixamo / AI 生成 / 外購）
2. Blender 批次轉換：
   ```powershell
   blender --background --python .ai/scripts/fbx_to_glb.py
   ```
3. 輸出到 `public/models/zombie_N/`：
   - `zombie_N.glb`（主 Mesh + 骨架，Draco 壓縮）
   - `zombie_N_idle.glb`（待機動畫）
   - `zombie_N_attack.glb`（攻擊動畫）
   - `zombie_N_hurt.glb`（受擊動畫）
   - `zombie_N_dying.glb`（死亡動畫）
   - `zombie_N_run.glb`（跑步動畫）

### 方案 B：複用現有模型（快速上線）

在 `heroes` 表設定 `modelId = 'zombie_X'`（X = 1~14），跳過模型製作。
英雄會使用已存在的 3D 模型，但有獨立的名稱/數值/技能。

## 七、常見陷阱

1. **effectId 不可重複** — 全表唯一，建議含 heroId 避免衝突
2. **statusValue 是小數** — 0.2 = 20%，不是 20
3. **poison 的 statusValue 是 maxHP 比例** — 0.03 = 3%（已有 HP cap 100,000 防 Boss 破格）
4. **burn / bleed 的 statusValue 是 ATK 比例** — 0.3 = ATK×30%
5. **skill_effects 必須有** — 沒有 skill_effects 連結，技能在 SkillDescPanel 不會顯示效果
6. **icon 欄位用 emoji** — 不要用文字 key，直接存 emoji 字元（🔥 ❄️ 💀 etc.）
7. **passive_trigger** 存在 skill_templates — 同時效果的 trigger_type 也要正確設定
8. **modelId 必須對應到 public/models/** — 否則 3D 模型載入失敗（可用方案 B 複用）
