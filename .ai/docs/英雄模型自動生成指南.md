# 英雄 3D 模型自動生成指南

> 版本：v1.0 ｜ 最後更新：2026-03-15
> 對應腳本：`.ai/scripts/generate_hero.mjs`

## 1. 概述

本指南說明如何**全自動**生成可直接在 Three.js 中使用的英雄 3D 模型與動畫。

### 完整管線

```
[輸入] ──→ [3D 模型生成] ──→ [自動綁骨] ──→ [動畫套用] ──→ [GLB 轉換] ──→ [大頭照]
  │          Meshy.ai        Mixamo         Mixamo         Blender       Puppeteer
  │         (text/image      (auto-rig)   (5 animations)  (Draco壓縮)
  │          → 3D mesh)
  │
  ├─ 文字描述 ("A zombie warrior with dark armor")
  ├─ 參考圖片 (reference.png)
  └─ 現有模型 (character.fbx)
```

### 輸出結構

```
public/models/zombie_N/
├── zombie_N.glb              ← Mesh + 骨架（Draco 壓縮 + JPEG 貼圖）
├── zombie_N_idle.glb         ← 待機動畫（~100-170 KB）
├── zombie_N_attack.glb       ← 攻擊動畫
├── zombie_N_hurt.glb         ← 受擊動畫
├── zombie_N_dying.glb        ← 死亡動畫
├── zombie_N_run.glb          ← 跑步動畫
└── thumbnail.png             ← 512×512 大頭照
```

## 2. 前置需求

### 2.1 軟體

| 軟體 | 版本 | 用途 | 安裝方式 |
|------|------|------|---------|
| Node.js | 18+ | 執行腳本 | `winget install OpenJS.NodeJS.LTS` |
| Blender | 3.6+ | FBX→GLB 轉換 | `winget install BlenderFoundation.Blender` |
| Puppeteer | (dev dep) | 大頭照生成 | 已包含在 `package.json` |

### 2.2 API 帳號

| 服務 | 用途 | 註冊 | 免費額度 |
|------|------|------|---------|
| **Meshy.ai** | AI 3D 模型生成 | https://app.meshy.ai/ | 200 credits/月 |
| **Mixamo** | 自動綁骨 + 動畫 | https://www.mixamo.com/ | 完全免費（需 Adobe 帳號） |

### 2.3 設定 API 金鑰

```powershell
# 複製設定檔範本
cp .ai/scripts/hero-gen.env.example .ai/scripts/hero-gen.env

# 編輯設定檔，填入你的 API 金鑰
notepad .ai/scripts/hero-gen.env
```

#### 取得 Meshy API Key

1. 登入 https://app.meshy.ai/
2. 點擊 Settings → API Keys → Create API Key
3. 複製 Key 填入 `MESHY_API_KEY`

#### 取得 Mixamo Token

1. 登入 https://www.mixamo.com/
2. 按 F12 開啟 DevTools → Network 分頁
3. 在 Mixamo 頁面上隨意瀏覽動畫
4. 在 Network 中找到任何 `www.mixamo.com/api` 開頭的請求
5. 從 Request Headers 複製：
   - `Authorization: Bearer <TOKEN>` → 填入 `MIXAMO_TOKEN`
   - `x-csrf-token: <CSRF>` → 填入 `MIXAMO_CSRF`

> ⚠️ Mixamo Token 有效期約 24 小時，過期需重新取得

## 3. 使用方式

### 3.1 方式一：文字描述生成（最自動化）

```powershell
node .ai/scripts/generate_hero.mjs `
  --id zombie_15 `
  --prompt "A zombie warrior with dark armor, glowing red eyes, and a massive sword"
```

**流程**：Meshy text-to-3D → Mixamo 綁骨 → 下載 5 動畫 → Blender 轉 GLB → 大頭照

**提示詞技巧**：
- 使用英文效果最佳
- 描述角色外觀特徵（盔甲、武器、體型等）
- 加入材質描述（金屬、皮革、發光等）
- 範例：
  - `"A hulking zombie brute with exposed muscles and bone armor"`
  - `"A spectral undead mage in tattered robes with ethereal glow"`
  - `"A nimble zombie assassin with blade arms and dark hood"`

**風格選項**（`--art-style`）：
| 風格 | 說明 | 適用場景 |
|------|------|---------|
| `realistic` | 寫實風格（預設） | 標準 3D 模型 |
| `cartoon` | 卡通風格 | 可愛/誇張造型 |
| `low-poly` | 低面數風格 | 效能優先 |
| `sculpture` | 雕塑質感 | 無貼圖/材質 |

### 3.2 方式二：參考圖片生成

```powershell
node .ai/scripts/generate_hero.mjs `
  --id zombie_15 `
  --image ./reference_art.png
```

**適用場景**：有角色設計圖或概念藝術，想轉成 3D

**圖片要求**：
- 支援 PNG / JPEG
- 建議正面全身照
- 背景越簡潔越好（白底最佳）
- 建議解析度 512×512 以上

### 3.3 方式三：已有模型

```powershell
# 有現成 FBX/OBJ，需要綁骨 + 動畫 + 轉換
node .ai/scripts/generate_hero.mjs `
  --id zombie_15 `
  --model ./my_character.fbx

# 模型已有骨架，跳過綁骨
node .ai/scripts/generate_hero.mjs `
  --id zombie_15 `
  --model ./rigged_character.fbx `
  --skip-rig
```

**適用場景**：從 Sketchfab / TurboSquid / itch.io 下載的模型

### 3.4 方式四：僅轉換

```powershell
# 已手動放好 FBX，只需 FBX→GLB 轉換
node .ai/scripts/generate_hero.mjs --id zombie_15 --convert-only
```

**前提**：`public/models/zombie_15/` 中已有：
- `idle.fbx`（含 Mesh + 骨架 + idle 動畫）
- `attack.fbx`、`hurt.fbx`、`dying.fbx`、`run.fbx`（各含對應動畫）

## 4. 手動 Mixamo 流程（備案）

如果自動化 Mixamo API 因 Token 過期或其他原因失敗，可以手動操作：

### Step 1: 上傳角色

1. 前往 https://www.mixamo.com/
2. 點擊 "Upload Character"
3. 上傳你的 FBX / OBJ 檔案
4. 等待自動綁骨完成
5. 調整骨骼定位（如需要）

### Step 2: 下載動畫

需要下載 **5 個動畫**，每個都選 **FBX Binary (.fbx)** 格式、**Without Skin**（不含 Mesh）：

| 動畫檔名 | Mixamo 搜尋關鍵字 | 推薦動畫 |
|----------|-------------------|---------|
| `idle.fbx` | "zombie idle" 或 "breathing idle" | Zombie Idle（含 Mesh，選 With Skin） |
| `attack.fbx` | "zombie attack" 或 "punch" | Zombie Attack / Punching |
| `hurt.fbx` | "hit reaction" | Hit Reaction / Getting Hit |
| `dying.fbx` | "zombie death" 或 "dying" | Zombie Dying / Death |
| `run.fbx` | "zombie run" | Zombie Run / Running |

> ⚠️ **idle.fbx 必須選 With Skin**（包含 Mesh + 骨架），其餘 4 個選 Without Skin

### Step 3: 放入資料夾

```
public/models/zombie_15/
├── idle.fbx      ← 含 Mesh（With Skin）
├── attack.fbx    ← 動畫 only（Without Skin）
├── hurt.fbx
├── dying.fbx
└── run.fbx
```

### Step 4: 轉換

```powershell
node .ai/scripts/generate_hero.mjs --id zombie_15 --convert-only
```

## 6. 技術細節

### 6.1 Meshy.ai API

**Text-to-3D（兩階段）**：
```
POST /openapi/v2/text-to-3d  (mode: "preview")  → 快速低精度模型
POST /openapi/v2/text-to-3d  (mode: "refine")   → 精細化高品質
GET  /openapi/v2/text-to-3d/{id}                 → 輪詢狀態
```

**Image-to-3D**：
```
POST /openapi/v1/image-to-3d  → 圖片轉 3D（支援 base64 data URI）
GET  /openapi/v1/image-to-3d/{id}  → 輪詢狀態
```

**輸出格式**：FBX / GLB / OBJ / USDZ（腳本優先下載 FBX 供 Mixamo 使用）

**費用**：Preview ~5 credits、Refine ~10 credits、Image-to-3D ~10 credits

### 6.2 Mixamo API（非官方）

```
POST /api/v1/characters/upload   → 取得 presigned URL
PUT  <presigned-url>             → 上傳檔案
POST /api/v1/characters          → 觸發自動綁骨
GET  /api/v1/characters/{id}     → 查詢綁骨進度
GET  /api/v1/products?type=Motion&query=<q>  → 搜尋動畫
POST /api/v1/exports             → 建立匯出任務
GET  /api/v1/exports/{id}        → 查詢匯出進度 + 下載連結
```

**認證**：使用 Adobe IMS Bearer Token（從瀏覽器 DevTools 取得）

### 6.3 Blender 轉換

使用既有的 `.ai/scripts/fbx_to_glb.py`：
- `global_scale=100.0`（抵消 FBX cm→m 縮放）
- Draco 壓縮（compression_level=6）
- 面數超過 15,000 自動 Decimate
- 貼圖超過 2048 自動縮小 + 轉 JPEG

### 6.4 前端整合

生成的 GLB 檔案與現有的 `zombie_1` ~ `zombie_14` 完全相同格式，前端載入器已原生支援：

```typescript
// src/loaders/glbLoader.ts — 自動載入 6 個 GLB
preloadHeroModel('zombie_15')  // → Promise.all([mesh, idle, attack, hurt, dying, run])

// src/components/ZombieModel.tsx — 直接渲染
<ZombieModel modelId="zombie_15" isPlayer={true} state="IDLE" />
```

**不需要修改任何前端程式碼**，只需在 Google Sheets 英雄表新增記錄即可。

## 7. 疑難排解

### Meshy 生成品質不佳

- 調整提示詞，加入更多細節描述
- 嘗試不同 `--art-style`（realistic / cartoon / sculpture）
- 使用 `--image` 搭配高品質參考圖片
- 免費方案有額度限制，超過需付費

### Mixamo Token 過期

Mixamo Token 約 24 小時過期，出現 401 錯誤時：
1. 重新登入 https://www.mixamo.com/
2. 從 DevTools Network 取得新 Token
3. 更新 `hero-gen.env`

### Mixamo 綁骨失敗

- 模型必須是**人形**（有頭、軀幹、四肢）
- 模型不能太複雜（面數建議 < 50,000）
- 必須是 T-pose 或 A-pose
- 嘗試轉為 OBJ 格式再上傳

### Blender 轉換失敗

- 確認 Blender 版本 ≥ 3.6
- 確認 FBX 檔案完整未損壞
- 檢查 `.ai/scripts/fbx_to_glb.py` 是否存在
- 嘗試手動執行：`blender --background --python .ai/scripts/fbx_to_glb.py -- --only zombie_N`

### 模型在遊戲中大小異常

- GLB 使用 Z-up 座標系，高度計算用 `bbox.max.z - bbox.min.z`
- Blender 匯入 `global_scale=100.0` 是必要的
- 如果模型太大/太小，檢查 FBX 的原始單位設定

### 動畫跑步時位移異常

- Mixamo 下載動畫時確認 "In Place" 選項勾選（消除 root motion）
- 腳本會自動過濾 `.position` track 來移除 root motion

## 8. 完整範例：從零到上線

```powershell
# === Step 1: 設定 API 金鑰（首次使用） ===
cp .ai/scripts/hero-gen.env.example .ai/scripts/hero-gen.env
# 編輯 hero-gen.env 填入 MESHY_API_KEY 和 MIXAMO_TOKEN

# === Step 2: 生成英雄模型 ===
node .ai/scripts/generate_hero.mjs `
  --id zombie_15 `
  --prompt "A hulking undead berserker with bone spikes protruding from shoulders, rotting green skin, and chains wrapped around arms"

# === Step 3: 確認輸出 ===
Get-ChildItem public/models/zombie_15/

# 預期輸出：
# zombie_15.glb          (~1-7 MB)
# zombie_15_idle.glb     (~100-170 KB)
# zombie_15_attack.glb
# zombie_15_hurt.glb
# zombie_15_dying.glb
# zombie_15_run.glb
# thumbnail.png          (512×512)

# === Step 4: 在 Google Sheets 新增英雄記錄 ===
# HeroID: 15
# ModelID: zombie_15
# Name: 骨刺狂戰士
# Type: 力量
# Rarity: SSR
# HP: 180, ATK: 55, DEF: 30, Speed: 7, CritRate: 10, CritDmg: 60

# === Step 5: 部署 Workers ===
cd workers; npx wrangler deploy; cd ..

# === Step 6: 啟動遊戲測試 ===
npx vite --host
# 在遊戲中抽到或選擇 zombie_15 即可看到新英雄
```
