# 大頭照（Thumbnail）生成指南

> 自動從 FBX 模型生成角色大頭照，用於遊戲 UI 的角色卡牌顯示。

## 1. 概述

遊戲中的角色選擇卡牌需要每個 zombie 的大頭照（正面半身裁切）。
本工具使用 **Puppeteer + Three.js** 離線渲染 FBX 模型，自動裁切頭部區域並輸出 PNG。

## 2. 檔案結構

| 檔案 | 說明 |
|------|------|
| `.ai/scripts/generate_thumbnails.js` | Node.js 主腳本 — 啟動本地 HTTP server、用 Puppeteer 開啟渲染頁面、截圖 |
| `public/thumbnail.html` | Three.js 渲染頁面 — 載入 FBX、自動對焦頭部、裁切輸出 |

### 輸出位置

```
public/models/zombie_X/
├── thumbnail.png    ← 512×512 PNG，透明背景，頭部裁切
├── idle.fbx
├── ...
```

## 3. 使用方式

### 3.1 前置需求

- Node.js 18+
- `puppeteer` 套件（`npm install puppeteer`）
- FBX 模型（`idle.fbx` 或任何含 idle 動畫的 FBX）

### 3.2 生成所有 zombie 的大頭照

```powershell
node .ai/scripts/generate_thumbnails.js
```

- 會掃描 `public/models/zombie_*/` 下所有資料夾
- 若 `thumbnail.png` 已存在則跳過
- 若要強制重新生成，先刪除 `thumbnail.png`

### 3.3 強制重新生成全部

```powershell
# 刪除所有現有大頭照後重新生成
Get-ChildItem "public\models\zombie_*\thumbnail.png" | Remove-Item
node .ai/scripts/generate_thumbnails.js
```

## 4. 技術細節

### 4.1 工作流程

```
Node.js 腳本
  │
  ├─ 1. 啟動 Express 靜態檔伺服器（port 自動分配）
  │     └─ 服務 public/ 目錄 + node_modules/（提供 Three.js）
  │
  ├─ 2. 啟動 Headless Puppeteer 瀏覽器
  │
  ├─ 3. 對每個 zombie 資料夾：
  │     ├─ 開啟 thumbnail.html?model=/models/zombie_X/idle.fbx&size=512
  │     ├─ 等待 window.__THUMBNAIL_READY__ === true
  │     ├─ 讀取 canvas.toDataURL('image/png')
  │     └─ 寫入 thumbnail.png
  │
  └─ 4. 關閉瀏覽器與伺服器
```

### 4.2 thumbnail.html 渲染邏輯

1. **載入模型**：使用 `FBXLoader` 載入指定 FBX
2. **播放 Idle 動畫**：避免擷取 T-pose，`mixer.update(0.6)` 推進到自然姿勢
3. **自動對焦頭部**：
   - 計算完整 bounding box
   - 瞄準身高 82% 位置（大約頭部）
   - 鏡頭距離 = `maxDim * 0.6`（近距離特寫）
4. **頭部裁切**：
   - 取 bounding box 上方 40%（身高 60%~100%）作為頭部區域
   - 將 8 個角點投影到螢幕座標
   - 裁切為正方形，等比縮放到 512×512
5. **透明背景**：WebGL `alpha: true` + `clearColor(0, 0, 0, 0)`

### 4.3 燈光設定

```javascript
// 半球光（環境光）
new THREE.HemisphereLight(0xffffff, 0x444444, 1.0)
// 方向光（主光）
new THREE.DirectionalLight(0xffffff, 1.0).position.set(1, 2, 1)
// 環境光（補光）
new THREE.AmbientLight(0xffffff, 0.3)
```

### 4.4 材質修正

FBX 匯入的貼圖需手動設定 `sRGBEncoding` 並觸發 `needsUpdate`，
否則顏色偏暗（Three.js 預設 LinearEncoding）。

## 5. 前端使用

大頭照在 `App.tsx` 的角色卡牌中使用：

```typescript
// 大頭照路徑
const thumbnailUrl = `${import.meta.env.BASE_URL}models/${zombieId}/thumbnail.png`

// 在 <img> 或 CSS background 中顯示
<img src={thumbnailUrl} alt={zombieId} />
```

## 6. 常見問題

### Q: 大頭照裁切位置不對
- `thumbnail.html` 中的 `sizeVec.y * 0.82` 控制瞄準高度（82% 身高）
- `sizeVec.y * 0.6` 控制裁切下邊界（身高 60% 以上為頭部區域）
- 可調整這兩個參數適應不同體型

### Q: 大頭照背景不透明
- 確認 WebGLRenderer 設定 `alpha: true` 和 `clearColor(0, 0)` 

### Q: Puppeteer 找不到 Chrome
- `puppeteer` 首次安裝會自動下載 Chromium
- 若公司網路受限，設定 `PUPPETEER_SKIP_DOWNLOAD=1` 再手動指定 Chrome 路徑

### Q: 生成的圖片是黑的 / 空白
- 確認 `node_modules/three/` 存在（thumbnail.html 透過 importmap 引入 Three.js）
- 確認 FBX 檔案路徑正確（需為相對於 `public/` 的路徑）

## 7. 新增 zombie 後生成大頭照

```powershell
# 1. 放入 FBX 模型到 public/models/zombie_N/
# 2. 生成大頭照（會自動跳過已有 thumbnail.png 的資料夾）
node .ai/scripts/generate_thumbnails.js
# 3. 確認 public/models/zombie_N/thumbnail.png 已產生
```
