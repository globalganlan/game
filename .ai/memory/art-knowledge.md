# 🎨 美術知識庫 — 設計自審規範與品質標準

> 建立日期：2026-02-27
> 角色：04-UI Design Agent
> 目的：確保所有 UI 與 Icon 設計維持**寫實、精緻**的統一風格，美術團隊主動自審

---

## 🔴 核心原則

> **「寫實・精緻・主動自審」**
>
> 美術的職責不只是畫圖交差。
> 每次完成 UI 或 icon 設計後，必須**主動檢視是否符合專案的美學標準**。
> 風格定位：**暗黑末日寫實風**，精緻度參考 3A 手遊水準。
> 若發現不符合標準，必須自行調整，不需要等使用者提出。

---

## 🎯 風格定位

### 整體方向

| 維度 | 標準 |
|------|------|
| 風格 | 暗黑末日寫實風（Dark Fantasy / Post-Apocalyptic Realism） |
| 精緻度 | 接近 3A 手遊水準，不可粗糙 |
| 色調 | 以 `--color-bg-primary: #1a0e06`（深棕黑）為基底 |
| 氣氛 | 末日、緊張、殘酷但有英雄主義的光芒 |
| 對比 | 深色背景 + 高飽和度重點色（金/橙/紅/藍） |

### 禁止風格

- ❌ 卡通 / Q 版 / 可愛風
- ❌ 粗線條 / 平面插畫風
- ❌ 過度扁平化（需有材質感、光影層次）
- ❌ 不協調的色彩（螢光色、高明度低飽和）
- ❌ 隨意使用 emoji 作為正式 UI 元素（emoji 僅作為開發期暫時方案）

## 🔍 自審 Checklist（每次設計交付前必做）

### 一、UI 元素

- [ ] **一致性**：新元素與現有 UI 風格統一（字體、圓角、陰影、色階）
- [ ] **材質感**：按鈕、面板、框線有足夠的紋理或光影，不是純色扁平
- [ ] **層次感**：前景 / 中景 / 背景有清晰的視覺層級
- [ ] **可讀性**：文字在深色背景上清晰可辨，對比度 ≥ 4.5:1
- [ ] **邊緣處理**：無毛邊、鋸齒、溢出、切邊
- [ ] **RWD 適配**：mobile / tablet / desktop 三端檢查佈局不破版

### 二、Icon 設計

- [ ] **寫實度**：icon 風格寫實（3D 渲染風 or 高精度手繪），禁止卡通/扁平
- [ ] **辨識度**：每個 icon 在 32×32 尺寸下仍能清楚辨識含義
- [ ] **統一性**：同類型 icon（技能/裝備/物品）使用統一的底框、光暈、色溫
- [ ] **色彩語意**：
  - 🔥 火系 → 紅橙漸層
  - 💧 水系 → 藍紫漸層
  - 🌿 地系 → 綠棕漸層
  - ⚡ 雷系 → 黃白漸層
  - ❄️ 冰系 → 冰藍白
  - 🌑 暗系 → 紫黑漸層
- [ ] **Buff/Debuff icon**：正面效果用暖色、負面效果用冷色/灰色
- [ ] **尺寸規範**：輸出 @1x / @2x / @3x，確保各解析度銳利

### 三、角色 & 模型表現

- [ ] **3D 模型**：角色在場景中不穿模、不漂浮、比例正確
- [ ] **大頭照**：英雄列表 thumbnail 清晰、光影一致、裁切合理
- [ ] **動畫品質**：idle/attack/hurt/dying 動畫自然流暢、無骨架扭曲
- [ ] **場景融合**：角色光影與場景 fog / lighting 一致

## 📝 歷史經驗

### Case 1：emoji 替代正式 icon（2026-02-27）

- **背景**：63 個技能需要 icon，開發階段使用 emoji 快速填入
- **問題**：emoji 在不同平台渲染不同，且不符合「寫實精緻」標準
- **決策**：emoji 作為暫時方案，前端 `resolveSkillIcon()` 可辨識並 fallback
- **後續**：待美術製作正式 icon 後替換 Sheet 資料 + 前端引用路徑

### Case 2：貨幣 icon 寫成 emoji 而非 CurrencyIcon 元件（2026-03-01）

- **背景**：開發 PWA 安裝獎勵 & 帳號綁定獎勵功能
- **問題**：SettingsPanel 中的獎勵預覽文字直接用 💎🪙 emoji，未使用 `<CurrencyIcon>` 元件
- **原因**：pwaService.ts 是純 TS 檔回傳 `string[]`，不能直接內嵌 JSX
- **修正**：將 PWA benefits 文字改由 SettingsPanel 直接渲染 JSX，綁定獎勵行同步改用 CurrencyIcon
- **規則**（ADR-007）：
  - 金幣 → `<CurrencyIcon type="gold" />`（金色圓形 G）
  - 鑽石 → `<CurrencyIcon type="diamond" />`（藍色菱形 D）
  - 經驗 → `<CurrencyIcon type="exp" />`（綠色方塊 E）
  - 星塵 → `<CurrencyIcon type="stardust" />`（黃色光暈圓形 S）
  - 其他道具 → `<ItemIcon itemId="..." />`（自動判斷貨幣 or emoji）
- **教訓**：任何 UI 顯示貨幣數量的地方，一律用 CurrencyIcon/ItemIcon，**絕對不可硬寫 emoji**
