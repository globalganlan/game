# 縮圖產生流程

目標：自動化把 `public/models/*` 底下的 FBX（以含 `idle` 的檔名為優先）渲染成上半身大頭照（透明背景），輸出為 `*_thumbnail.png` 存回對應模型資料夾。

概要：本專案已新增腳本與頁面，用來在 headless Chromium 中載入 FBX、應用 idle 動作（若有）、調整相機並裁切只保留頭部，最後把 PNG 存回模型資料夾。

檔案（已新增）：
- `scripts/generate_thumbnails.js` — Node 執行器（使用 `puppeteer` 啟動 headless Chromium，啟動一個靜態伺服器並載入 thumbnail 頁面）。
- `public/thumbnail.html` — 瀏覽器端渲染/裁切邏輯（使用 `three` 與 `FBXLoader`，播放 idle 動作、材質處理、裁切頭部並輸出透明背景）。

先決條件
- Node.js（專案發現使用 Node v18，但 puppeteer 會下載 Chromium）
- 已在專案目錄執行 `npm install` 以安裝相依（若未安裝，請先執行）。

安裝（如尚未安裝）
```bash
npm install
```

執行（一次性產生全部模型縮圖）
```bash
npm run generate:thumbnails
```

行為細節
- 腳本會掃描 `public/models/` 下的資料夾，對每個名稱符合 `zombie`（目前用於 zombine 系列）的資料夾尋找 FBX 檔（優先匹配 `*idle*.fbx`，若無則取第一個 `.fbx`）。
- 對每個模型，腳本會啟動一個小型靜態伺服器（對 `public/` 提供服務），用 `puppeteer` 開啟 `thumbnail.html?model=/models/<name>/<file>&size=512`。
- 頁面端會載入模型、嘗試播放名為 `idle` 的動畫（若動畫存在），並在模型 pose 應用後計算 bounding box、把相機對準上半身，render 成 canvas，再以 head-region 計算裁切方框，輸出為正方形 PNG（透明背景）。
- 最終檔名：`<folder>/<folder>_thumbnail.png`（例如 `public/models/zombie_1/zombie_1_thumbnail.png`）。

自定義
- 修改輸出尺寸：編輯 `scripts/generate_thumbnails.js` 中的 `size=512` 或在 `thumbnail.html` 讀取 `size` query。
- 調整裁切邏輯或相機位置：編輯 `public/thumbnail.html`，關鍵段落在載入模型後計算 bounding box、計算 `upperCenterY` 與 `distance`，以及裁切邏輯（使用 box 的 top 與 lower fraction）。

常見問題
- Chromium 下載或 puppeteer 啟動失敗：確保系統可連外或安裝 puppeteer 的可執行 Chromium，或改用 `PUPPETEER_SKIP_DOWNLOAD=1` 並提供 Chrome 可執行路徑。
- 頁面出現 404（如 `favicon.ico`）：這不影響縮圖結果，可忽略或放一個小 favicon 到 `public/`。
- 模型沒有貼圖或貼圖看起來偏暗：`thumbnail.html` 已嘗試將貼圖設為 sRGB encoding 並使用 physicallyCorrectLights，但若仍怪異可手動調整 light 強度或材質參數。
